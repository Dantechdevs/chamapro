Security Practices

How we protect your Chama's financial data.

Password Hashing

All passwords are hashed using Django's PBKDF2 algorithm with SHA-256. Plain-text passwords are never stored.

Role-Based Access Control

Admins, treasurers, secretaries, and members each see only what their role permits. No accidental data exposure.

CSRF Protection

Every form submission is protected against Cross-Site Request Forgery attacks using Django's built-in token system.

HTTPS Everywhere

All traffic is encrypted in transit using TLS 1.3. HTTP requests are permanently redirected to HTTPS.

Database Encryption

Sensitive fields are encrypted at rest. Database backups are encrypted and stored in separate locations daily.

Audit Logging

All critical actions — contributions, loans, role changes — are logged with timestamps for full accountability.
